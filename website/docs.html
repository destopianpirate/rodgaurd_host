Detection Flow

Step-by-step process every 0.5 seconds

1

📸 Continuous Capture

Pi v2 camera captures 640×480 road image. NEO-M8N reads GPS coordinates. MPU-9250 reads 9-axis motion data.

2

🧠 AI Analysis (YOLOv8)

Frame fed to YOLOv8 neural network. Detections with ≥50% confidence accepted. Severity calculated from bounding box area. IMU spikes supplement visual detection.

3

🤝 Collaborative Verification

Server checks if pothole exists within 30m. New → creates record. Existing → increments verification. No detection at known location → marks resolved.

4

🔔 Driver Alerts

Pothole → triple quick beep ⚡⚡⚡. Rough road (IMU) → double long beep 🔔🔔. 3-second cooldown prevents spam.

5

📊 Dashboard Display

Web dashboard polls server every 60s. Google Maps shows color-coded severity markers: 🔴 Critical · 🟠 High · 🟡 Medium · 🟢 Low

Quick Setup Guide

Get RoadGuard running in minutes

🍓 Raspberry Pi Setup

# Enable interfaces via raspi-config
sudo raspi-config
# Enable: Camera, I2C, Serial Port

# Copy files & install deps
cd /home/pi/roadguard
pip install -r requirements.txt

# Edit config.py with your server IP
# Start detection
python main.py

💻 Laptop Setup (Server + Dashboard)

# Terminal 1: Start Flask server
cd server
pip install -r requirements.txt
python app.py

# Terminal 2: Start website
cd website
python -m http.server 8000

# Open http://localhost:8000

API Reference

Flask REST API endpoints on port 5000

POST/api/potholesReport or verify a pothole
GET/api/potholesGet all potholes (filter: ?status=active&severity=high)
GET/api/potholes/nearbyGet potholes near GPS point (?lat=...&lng=...&radius=150)
POST/api/potholes/:id/resolveMark a pothole as resolved
GET/api/statsDashboard statistics summary
POST/api/seedGenerate demo data for testing
POST/api/clearWipe the entire database
GET/api/healthServer health check
